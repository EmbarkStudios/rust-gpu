pub struct EntityOrientedDenseMap<K: EntityOrientedMapKey<V>, V> { /* private fields */ }
Expand description

Map with K keys and V values, that is:

  • “entity-oriented” K keys, i.e. that are or contain exactly one entity (supported via K: EntityOrientedMapKey<V> for extensibility)
  • “dense” in the sense of few (or no) gaps in (the entities in) its keys (relative to the entities defined in the corresponding EntityDefs)

By design there is no way to iterate the entries in an EntityOrientedDenseMap.
