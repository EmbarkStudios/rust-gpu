Expand description

NoExpand indicates literal byte string replacement.

It can be used with replace and replace_all to do a literal byte string replacement without expanding $name to their corresponding capture groups. This can be both convenient (to avoid escaping $, for example) and performant (since capture groups donâ€™t need to be found).
